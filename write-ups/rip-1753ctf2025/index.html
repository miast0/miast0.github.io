Rest In Peace 1753CTF2025

A writeup of Forensics Rest In Peace challenge from 1753CTF 2025

13 April, 2025

  • ctf
  • writeup
  • forensics

Shout out to @jxkxxl for starting off this challenge and getting the bulk of the pre-work done. All I did was find the flag at the end.

Rest In Peace

Challenge Description

One of our agents never returned from his mission. We’ve found that he died captured by enemies. All that we got left are these backup files. Not sure how to get to them, but I think he used one of his family member names to secure it.

Initial Analysis

  • Objective: Recover data from backup files.
  • Obstacle: Files are secured, likely requiring a password.
  • Hint: Password is a family member’s name.

File Structure Analysis

  • Listing the directory contents revealed data/, snapshots/, keys/, index/ directories and a config file.
  • This structure is typical of a restic backup repository.
  • Attempting to read the config file indicated it’s binary, further supporting the restic hypothesis.

Likely Solution Path

  1. Identify the tool: The backup system is confirmed to be restic.
  2. Identify the protection: restic repositories are encrypted with a password.
  3. Find potential passwords: Need to locate a list of the agent’s family member names. This is the current unknown.
  4. Access the repository: Use restic commands (like restic -r . snapshots or restic -r . restore) along with the correct family name as the password to access and restore the backup contents.

Next Steps

  • Install restic if not present. (Installation command run)
  • Find the list of potential family member names.
  • Attempt to access the restic repository using the names.

Password Cracking

  1. Identify Key File: Located the key file in the keys/ directory: keys/c94dc6505d66ddc880fa4a2367461e5e4702b52312018921050459d8bd092fcf.
  2. Extract Hash: Used restic2john.py (from John the Ripper suite) on the repository directory (.) to extract the password hash: 
    python /usr/share/john/restic2john.py .
    Resulting hash: $restic$scrypt*32768*8*7*...*... (Full hash omitted for brevity)
  3. Save Hash: Saved the extracted hash to hash.txt.
  4. Crack Hash: Used john with its default wordlist to crack the hash: 
    john hash.txt
  5. Password Found: john successfully found the password: Christopher.

Data Restoration

  1. List Snapshots: Used the cracked password with restic to list snapshots: 
    restic -r . --password-command "echo Christopher" snapshots
    Found one snapshot: cd3921b4 containing /flag.
  2. Restore Snapshot: Restored the snapshot to the ./restore directory: 
    restic -r . --password-command "echo Christopher" restore cd3921b4 --target ./restore
  3. Examine Restored Files: Found flag1.txt and flag5.txt in the restore/ directory.

Finding a false flag

  • Contents of restore/flag1.txt: 1753c{fake
  • Contents of restore/flag5.txt: _flag}
  • Combined Flag: 1753c{fake_flag}
